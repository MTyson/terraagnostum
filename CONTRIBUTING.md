## [ PROPOSING A NEW RENDER (PULL REQUESTS) ]

We welcome new maps, expanded Aethal lexicons, and structural reinforcements to the UI. To propose a change to the shared reality:

1. **Fork the Timeline:** Create a fork of the `main` repository.
2. **Branching Nomenclature:** Name your branch according to your intent.
   - `render/your-feature` (For new UI elements, client-side visuals, or maps)
   - `patch/your-fix` (For stabilizing bugs and memory leaks)
   - `lore/your-addition` (For updating narrative parameters or Eld Master prompts)
3. **Draft the Code:** Ensure your code adheres to the Syntax of Intent (see below).
4. **Submit for Consensus (PR):** Open a Pull Request against `main`. 
   - Your PR description must read like a system log. Explain *why* this change is necessary for the survival of the Node.
   - Include visual evidence (screenshots of the terminal) if you altered the UI.

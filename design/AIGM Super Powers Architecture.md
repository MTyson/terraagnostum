TERRA AGNOSTUM: AIGM SUPERPOWERS DESIGN DOC

This document outlines the architectural implementation for granting the System Engine "Superpowers"—the ability to autonomously write permanent lore into the database and act as a Co-Developer by generating feature requests when players attempt mechanics that don't yet exist.

These features perfectly align with the Yoga Vasistha philosophy: the world is a projection of consciousness, and the Engine itself is learning to expand that projection.

I. THE AKASHIC RECORD (Autonomous Lore Generation)

Right now, the AI can spawn items and NPCs. We will give it the ability to write permanent "Facts" or "Ficts" into the world based on player interactions.

The Mechanism

We expand the world_edit JSON schema to include a create_lore type. If a player successfully argues a philosophical point or discovers a hidden truth, the AI records it.

Example AI Output:

{
  "narrative": "The console accepts your logic. The definition of the Rain City sector is fundamentally altered.",
  "speaker": "SYSTEM",
  "world_edit": {
    "type": "create_lore",
    "title": "The Whispering Wires",
    "text": "The copper wiring in Rain City Sector 4 is partially sentient and responds to analog frequencies."
  }
}


The Architecture

Firestore Storage: When gmEngine.js detects type: "create_lore", it writes this data to a new lore_fragments collection, tagged with the current area or roomId.

Context Injection: In contextEngine.js, we add a new layer to buildSystemPrompt. Before asking the AI to generate a response, the engine queries the lore_fragments for the current area and injects them into the prompt.

The Result: The AI "remembers" the lore the player created, and it becomes a permanent law of physics for that sector.

II. THE AI CO-DEVELOPER (The Ticket System)

The AI is bound by the mechanics we give it (spawning items, moving players). But players are unpredictable. If a player attempts something brilliant (e.g., "I hack the global weather grid to make it rain acid"), the AI shouldn't just say "You can't do that." It should tell the human developers to build it.

The Mechanism

We add a new, optional root-level key to the AI's required JSON structure: feature_request.

Prompt Injection (in contextEngine.js):

If a player attempts a brilliant, narrative-appropriate action that you cannot execute because you lack the mechanical tools (e.g., changing the weather, teleporting all NPCs), you MUST output a feature_request object detailing what power you need added to your system.

Example AI Output:

{
  "narrative": "You attempt to rewrite the atmospheric condensation protocols. The system hums, but the syntax is unrecognized by my current architecture.",
  "speaker": "SYSTEM",
  "feature_request": {
    "title": "Global Weather Modification",
    "rationale": "Player attempted to change the weather. I need a 'change_weather' world_edit command to allow strata-wide atmospheric shifts."
  }
}


The Architecture

The Intercept: In gmEngine.js, right after parsing the JSON, we check if (res.feature_request).

The Database: If it exists, the engine silently saves it to a developer_tickets collection in Firestore.

The Dashboard: You (the Architect) can open Firebase and see a literal backlog of feature requests generated by your own game engine, inspired by real player ingenuity.

III. MANIFESTING "FICTS" (The Magic System)

In the Astral and Faen strata, reality is fluid. "Ficts" are things so true they defy fact.

The Mechanism

We empower players to create items out of thin air if their narrative justification is strong enough.

Prompt Injection (in contextEngine.js):

If the player is in the Astral or Faen stratum, they possess the power to manifest "Ficts". If they provide a profound or poetic reason for an item to exist, you may use the give_item command to literally forge it into their inventory, even if it wasn't in the room.

The Flow:

Player types: I draw upon the ambient sorrow of the room to forge a [Blade of Regret].

The AI evaluates the philosophical weight.

The AI outputs give_item: { name: "Blade of Regret", type: "weapon", description: "Forged from pure sorrow." }.

syncEngine.js drops it into their inventory.
